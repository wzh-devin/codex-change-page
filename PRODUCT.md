# Product

## Register

product

## Users

Codex Desktop users on Apple Silicon Macs who want to personalize the app without modifying the official application bundle. They need a guided, reversible workflow for installing a theme engine, creating themes from personal images, applying or pausing a skin, diagnosing failures, and restoring the official appearance.

## Product Purpose

Codex 换肤助手 turns the existing Codex Dream Skin scripts into a trustworthy desktop product. Success means a first-time user can inspect compatibility, create a backup, choose an image, preview and apply a theme, verify the live result, and completely restore Codex without using Terminal or understanding CDP.

## Brand Personality

Calm, capable, and transparent. The interface should feel like a focused macOS creative utility: familiar enough to trust immediately, polished enough to make theme creation enjoyable, and explicit about every security-sensitive action.

## Anti-references

- Not a neon cyberpunk control panel competing with the user's theme artwork.
- Not a promotional landing page disguised as a desktop tool.
- Not a glass-card dashboard with decorative metrics and oversized gradients.
- Not a terminal wrapper that exposes raw scripts, logs, or unexplained technical jargon as the primary workflow.
- Not an installer that silently restarts Codex, changes API configuration, or hides restoration behavior.

## Design Principles

1. **Make the safe path the easy path.** Inspect, back up, preview, and confirm before applying.
2. **Show the user's theme, not the manager's decoration.** Artwork is the strongest visual element.
3. **Explain state plainly.** Codex, engine, skin, backup, and CDP status must never be ambiguous.
4. **Keep destructive actions reversible and separated.** Pause, restore, uninstall, and clear data are distinct operations.
5. **Earn trust through evidence.** Diagnostics and verification report what was checked and what failed without claiming success prematurely.

## Accessibility & Inclusion

- Target WCAG 2.2 AA contrast for text and controls.
- Support keyboard navigation, visible focus, VoiceOver labels, reduced motion, increased contrast, and system light/dark appearance.
- Never rely on color alone for status; pair semantic colors with icons and text.
- Keep animation between 150–250 ms and remove nonessential transitions when reduced motion is enabled.
